# PROJECT_TIMELINE

## Planning Assumptions
- Sprint duration: 2 weeks
- Team starts implementation in Sprint 3
- Parallel streams: Platform, Core Product, Integrations, AI, QA/SRE

## Timeline (Sprints 3-14)

| Sprint | Focus | Outcome |
|---|---|---|
| 3 | Authentication + Audit base | Secure identity baseline + audit framework |
| 4 | Organizations + Users/Teams | Tenant and membership foundation |
| 5 | Workspaces/Projects + Admin policies | Workspace-scoped control plane |
| 6 | Integrations foundation | Connector/OAuth/sync framework live |
| 7 | Google Ads + Meta + GA4 | First connector wave ingestion |
| 8 | Campaign domain + Analytics ingestion | Core marketing + canonical metrics |
| 9 | Reports and dashboards | Reporting MVP and exports |
| 10 | AI Assistant phase 1 | Prompt/context/streaming usable |
| 11 | Notifications + alerting | Operational communication layer |
| 12 | Billing + entitlements | Monetization baseline |
| 13 | AI Agents phase 1 | Orchestrated agent workflows |
| 14 | Production hardening | Reliability, security, scale readiness |

## Milestones
- MVP Functional Milestone: end of Sprint 9
- AI Assisted Milestone: end of Sprint 10
- Monetization Milestone: end of Sprint 12
- Go-live Hardening Milestone: end of Sprint 14

## Estimated Effort
- Total engineering effort: ~120 to 150 engineer-weeks
- QA/SDET effort: ~20 to 28 engineer-weeks
- DevOps/SRE effort: ~16 to 24 engineer-weeks
- Product/Design/TPM effort: ~24 to 32 person-weeks

## Critical Path
1. Auth/Org/Workspace
2. Integration foundation
3. Analytics canonical model
4. Reporting MVP
5. Hardening and operational readiness

## Contingency
- Reserve 15% capacity per sprint for unknown integration/API changes.
- Add one stabilization sprint if connector reliability goals are missed.
